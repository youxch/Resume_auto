## 工作流程

### 第一步：获取JD信息

如果用户只给了公司+岗位名（没贴JD），先问他："能把岗位JD贴给我吗？没有的话告诉我岗位名称和公司，我来判断方向。"

如果用户已给JD，直接进入分析。

### 第二步：判断方向

读取JD，判断岗位属于哪个方向：

**方向A — 天线/RF/电磁工程**
关键词：天线设计、射频、电磁仿真、超构表面、MIMO、波束赋形、天线测试、CMA、HFSS/CST

**方向B — AI算法/大模型**
关键词：深度学习、机器学习、生成模型、大模型、AI4S、仿真加速、逆向设计、科学计算、数据驱动

**混合型** — JD同时包含两方向（如Ansys、CST/达索系统、仿真软件AI化岗位）：选取两者最强的交叉点

判断后明确告诉用户："这个岗位是**方向X**，我将用对应策略定制。"

### 第三步：选定内容策略

读取 `references/project-details.md`，根据方向选择：

**方向A策略：**
- 项目排序：华为WiFi AP项目（CMA+工程交付）→ NUDT合作（RCS隐身+模式基因）→ NSFC（作为"AI辅助设计工具"背景补充）
- 论文：TAP已接受（近场全局相关+宽带多模追踪）→ Wiley论文（逆向设计，偏工程）
- 强调：CMA分析、超构表面设计、物理样品制备+第三方检测、工程交付、仿真工具（HFSS/CST）
- 技能关键词：特征模式分析、超构表面设计、宽带天线、方向图综合、MIMO、PSO优化、RCS缩减

**方向B策略：**
- 项目排序：NSFC（VAE逆向设计+并行计算）→ NUDT（DCVAE+55000条数据库）→ 华为大模型（CLIP架构）
- 论文：Wiley论文（深度学习逆向设计）→ ICMMT会议（GitHub首篇开源）→ TAP（作为"物理约束验证"背景）
- 强调：VAE/生成模型架构、大规模数据集、300x加速、逆向设计1秒、GitHub开源影响力（10万浏览）
- 技能关键词：深度学习、生成式AI、多模态大模型（CLIP）、科学计算加速、逆向设计、AI4S

**混合型策略：** 突出"我是用AI工具做电磁仿真的人" — 强调对仿真软件的深度理解（用HFSS/CST做博士研究）+ ML加速框架。开头写：*"长期使用HFSS/CST进行天线仿真研究，同时开发了基于深度学习的仿真加速框架，提速1000x以上。"*

### 第四步：生成定制简历内容

**先读 `references/style-guide.md`，然后按以下结构输出。**

节顺序（默认，可按方向微调）：
```
求职意向（一行，按方向写）
教育背景
科研项目（核心，最长最详细）
专业技能
个人荣誉
发表论文
```

**每个科研项目必须用 4 段式结构（不能简化成普通bullet）：**

```
[时间段]    [项目名称]（[来源：国家自然科学基金/国家重点研发计划/某重点单位]）主要负责人

1. 研究目标：针对[传统方法的瓶颈]，首次提出[创新范式/方法]，[效果]。

2. 研究内容：①… ②… ③… ④…

3. 关键结果：
   [子任务A]：[说明句，关键数字用**加粗**]
   [子任务B]：[说明句，关键数字用**加粗**]

4. 研究成果：发表[级别]期刊/会议论文X篇，获[奖项]，[其他补充]。
```

**写作规则（每条都是必须的）：**
- "首次"是核心词，每个创新点前必须有"首次"标注
- 创新对比句：「区别于[传统方法]，首次提出[你的方法]，实现[量化突破]」
- 全文无"我"字，动词直接开头：提出/构建/搭建/实现/突破/制备/验证
- 数字格式：300倍、0.012%、1秒、5000倍——对比格式写：「相比X，提升Y」
- NUDT项目统一写"某重点单位合作项目"，绝不写"国防科技大学"
- 论文后括注格式：（中科院1区TOP，第一作者，唯一学生作者）

**专业技能写宣言式段落，不是简单列表：**

方向A时：突出 精通电磁仿真（HFSS/CST）+ 精通天线测试 + 数理基础
方向B时：突出 精通AI4S研究范式（深度学习+高性能计算）+ 开源影响力 + 数理基础

### 第五步：公司定制备注

在简历内容之后，额外输出一段"**针对[公司名]的定制要点**"，说明：
1. 这份简历特别强调了什么（对应JD的哪些关键词）
2. 面试准备建议（1-2条）：比如"他们做HFSS，你可以提自己是深度用户"
3. 如果有遗漏数据（如FDA论文实验数据尚未填写），标记出来："注意：FDA论文草稿有12处待填实验数据，建议投递前更新"

## GitHub 部署子流程

### 执行步骤

**第一步：将目标网页版简历复制为 index.html**

```bash
cd "D:/YXC/YXC_know/Resume"
python -c "import shutil; shutil.copy('简历/尤栖冲_[公司]_[方向]_网页版.html', 'index.html')"
```

**第二步：提交并推送**

```bash
git add index.html
git commit -m "update: 首页换成[公司]_[方向]版本"
git push
```

推送后 GitHub Pages 约 1–2 分钟自动重新部署，访问 `https://youxch.github.io/Resume_auto` 即可查看。

### 首次启用 GitHub Pages（仅需一次）

若 Pages 尚未开启，告知用户：
1. 打开 `github.com/youxch/Resume_auto/settings/pages`
2. Source → 选 `Deploy from a branch`
3. Branch → 选 `main`，文件夹选 `/ (root)`
4. 点 Save，等待 1–2 分钟
